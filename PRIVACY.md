# ClassSense 隐私与合规声明

> 本文件是 ClassSense（AI 课堂读空气）项目的**隐私影响评估（PIA）/ 数据处理说明 / 安全响应流程**合集，
> 面向答辩评委、潜在部署方（学校/教研机构）、家长与学生、以及系统运维者。
> 相关技术实现请对照 `app/privacy.py`、`app/security.py`、`app/runtime_config.py` 的 `privacy_mode`。

---

## 1. 设计原则

ClassSense 面向课堂场景且涉及未成年人，我们将隐私作为**和功能扩展平行的第一主线**，遵循以下原则：

| 原则 | 具体落实 |
|---|---|
| 数据最小化 | 只保存姿态关键点 + 聚合计数，**从不保存原始视频** |
| 本地优先 | 姿态检测完全在本机 GPU/CPU 跑；LLM 支持本地 vLLM 部署 |
| 去标识化 | 不做人脸识别、不做 Re-ID；学生只用"本节课内匿名 track_id"，下课即销毁 |
| 目的限定 | 数据只用于教学改进，严禁用于评比、排名、扣分 |
| 反身效应规避 | **不提供学生退出开关**（见 §2.3）|
| 最短保留期 | 快照 30 天、报告 1 学期后自动清理 |
| 可解释 | 每条 AI 判定保留规则/prompt，支持事后质询 |

## 2. 数据处理规范

### 2.1 采集范围

- **视觉数据**：摄像头画面仅驻留内存，经 YOLOv8-Pose 提取 17 关键点后即丢弃
- **结构化数据（落盘）**：每 3 秒一条快照，包含：
  - 该时刻总人数
  - 聚合专注度分数（0-100）
  - 行为计数（`focused/head_down/lying_down/hand_raised/looking_away` 各多少人）
  - **不包含**座位坐标、不包含具体身份、不包含原始像素

### 2.2 上传外部 LLM 的数据

系统支持外部 LLM（DeepSeek、智谱、通义、本地 vLLM）。发给外部 API 的 **prompt 只包含聚合数字**，示例：

```
当前课堂数据（已上课 15 分钟）：
- 总人数：32
- 实时专注度：78%
- 本阶段平均专注度：74%
- 趋势：平稳
- 行为分布：专注25人、低头4人、举手3人
```

**绝不上传：**原始画面、关键点坐标、学生 ID、座位位置、摄像头编号。

### 2.3 为什么**不**提供学生退出开关

这是一个经过深思的决定。看似"赋予学生自主权"，实则会触发**反身效应（chilling effect）与霸凌风险**：

- 退出者在系统中呈现为缺失信号，**老师能在监视页面直接察觉"XXX 关闭了采集"**
- 这种可见性会演变为：
  - 老师将退出者列入"不配合学生"名单，进行隐性针对
  - 同学间形成"老实采集 vs 不老实的"二元对立，助长霸凌
  - 家长之间互相攀比"你家孩子退出了说明有问题"

**我们的替代方案**是把整条数据管线改造成"对全员同样温和"：

1. **全局隐私模式开关**：老师一键打开（公开课/敏感场景）→ 全员马赛克 + 全员不落盘 + 全员不调 LLM，不针对个人
2. **座位而非个人的粒度**：纵向画像只能到"A3 座位"，从数据结构层面阻断个人跟踪
3. **禁止排名类视图**：系统在 prompt 和 UI 层硬编码"不输出针对具体学生的批评建议"
4. **Opt-out 走学校流程**：如家长书面要求，由学校通过行政手段禁用该班级系统，而非学生个人在运行时退出

### 2.4 保留期与销毁

| 数据类型 | 保留期 | 销毁方式 |
|---|---|---|
| 实时内存帧 | < 33 ms | 推理结束立即覆盖 |
| 快照（SQLite） | ≤ 30 天 | 定期清理任务 + 设置页手动清空 |
| 课堂报告 | 一个学期 | 学期末自动归档并清空原库 |
| LLM 交互日志 | 与快照同步 | 含 prompt/output，便于质询 |
| 调试预览帧 | 仅按需生成 | 不落盘 |

## 3. 聚合数据发布规则（差分隐私 + k-匿名）

发布"班级/年级/学科"等聚合统计时，必须调用 `app/privacy.py` 提供的工具：

```python
from app.privacy import safe_group_stat

# 发布某座位区平均专注度
value = safe_group_stat(
    count=8,            # 该分组样本量
    value=74.5,         # 真实均值
    sensitivity=100.0,  # 百分比数据的敏感度上限
    epsilon=1.0,        # 隐私预算（越小越严格）
    lo=0.0, hi=100.0,
)
# → count < 5 时返回 '*'；否则返回加了 Laplace 噪声的数字
```

**硬性规则：**
- `K_MIN = 5`：任何分组样本量少于 5 不发布具体数值
- `epsilon = 1.0` 默认；敏感发布（如学科对比）可降至 0.5
- 裁剪到 `[0, 100]` 防止噪声造成明显越界

## 4. 安全工程

### 4.1 凭据加密

- `llm_config.json` 中的 `api_key` 使用 `app/security.py` 加密落盘
- 机器密钥存于项目根目录 `.secret_key`（`.gitignore` 已排除、权限 600）
- 加密方案：HMAC-SHA256 派生流密钥 + XOR + HMAC 完整性标签
- **首次启动自动迁移**：若检测到明文 api_key，立即加密并回写
- 所有密文前缀 `enc:v1:`，便于版本升级

### 4.2 网络传输

- 默认监听 `127.0.0.1`（不暴露到局域网外）
- 生产部署**必须**反向代理为 HTTPS + WSS（示例 Nginx 配置见 README 补充）
- 未配置 TLS 的实例不应处理真实课堂数据

### 4.3 数据库加密（推荐升级项）

- 当前 SQLite 未加密
- 生产环境建议替换为 SQLCipher 封装，或使用磁盘全盘加密（BitLocker / FileVault）

### 4.4 数据泄露响应流程

一旦怀疑数据泄露，按以下时限执行：

| 时间窗口 | 行动 |
|---|---|
| T+1h | 下线服务、保留日志快照、更换所有外部 API Key |
| T+24h | 评估影响范围（哪些班级、哪段时间、是否含 prompt 日志）|
| T+72h | 通报学校信息化主管 + 班主任 + 监护人；提交事故报告 |
| T+7d | 复盘并修补；如有外部 API 泄露，回收并吊销 |

## 5. 可解释性 / 质询支持

每一条 AI 判定都附带可审计的依据：

- **规则层**：行为判定（低头/趴桌/举手）在 `behavior_analyzer.py` 中有明确阈值，报告页展示"该行为因为：鼻眼可见度 0.42 + 头部俯角 52°"
- **LLM 层**：Agent 每条分析都在 `agent_manager.py` 的 `history` 中保留：
  - `prompt_system`：系统指令原文
  - `prompt_user`：注入的聚合上下文
  - `content`：LLM 完整回复
  - `timestamp`：生成时间
- 家长或督导若对某条评价有质疑，可凭报告 ID + 时间戳调取完整 prompt 链路复盘

## 6. 反滥用红线

以下使用方式**明确违反本项目设计意图**，系统在 prompt 和 UI 层都会拒绝：

- ❌ 展示学生专注度排名
- ❌ 输出"建议处罚/批评 XXX"
- ❌ 将专注度接入期末评分
- ❌ 摄像头未经告知开启（见 §7）
- ❌ 把 LLM prompt 中加入学生真实姓名

## 7. 告知与同意

### 7.1 摄像头可见性

- 硬性要求：摄像头工作期间，教室应有**明显的红色指示灯或屏幕提示**"本节课正在进行专注度辅助分析"
- 软件层面已在 WebSocket 实时广播"课堂进行中"状态，前端显著位置展示

### 7.2 知情同意模板（简化版）

```
本班级使用 ClassSense 课堂专注度辅助分析系统。
系统不识别身份、不录制视频、不接入学生评分。
数据保留期：快照 30 天，报告 1 学期。
如有疑问或撤回要求，请联系：<班主任邮箱>
```

完整模板见 §9。

### 7.3 法规对标

- 🇨🇳 《个人信息保护法》：未成年人数据属"敏感个人信息"，需取得监护人单独同意
- 🇨🇳 《未成年人保护法》：教学场景数据处理应符合"儿童最大利益原则"
- 🇨🇳 《数据安全法》：本地化部署 + 最小化 + 事故报告机制
- 🇪🇺 GDPR Art. 8（参考）：16 岁以下需父母同意
- 🇺🇸 FERPA（参考）：如涉外部署需另行评估

### 中文版

```
尊敬的家长、同学：

本班级在 <学期/时段> 期间使用 ClassSense 课堂专注度辅助分析系统。

```
